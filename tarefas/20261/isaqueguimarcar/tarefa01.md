## f) Branches e Pull Request utilizando GitFlow

O GitFlow é um modelo de organização de branches bastante utilizado no desenvolvimento colaborativo.

As principais branches são:

- **main:** contém a versão estável do projeto.
- **develop:** reúne as funcionalidades em desenvolvimento.
- **feature:** utilizada para implementar novas funcionalidades.
- **release:** prepara uma nova versão para publicação.
- **hotfix:** utilizada para correções urgentes na versão de produção.

Após concluir uma funcionalidade em uma branch `feature`, é criado um **Pull Request** para que o código seja revisado antes de ser integrado às branches principais, garantindo maior qualidade e segurança no desenvolvimento.

## g) Versionamento Semântico

O Versionamento Semântico (SemVer) utiliza o padrão **MAJOR.MINOR.PATCH**.

- **MAJOR:** alterações incompatíveis com versões anteriores.
- **MINOR:** novas funcionalidades compatíveis.
- **PATCH:** correções de erros que não alteram funcionalidades.

Esse padrão facilita o controle das versões do software e permite compreender rapidamente o impacto de cada atualização.

---

## h) Conventional Commits

Conventional Commits é uma convenção para padronizar mensagens de commit.

Alguns exemplos são:

- `feat:` adiciona uma nova funcionalidade;
- `fix:` corrige um erro;
- `docs:` altera a documentação;
- `style:` modifica apenas formatação;
- `refactor:` refatora o código;
- `test:` adiciona ou modifica testes;
- `chore:` tarefas de manutenção.

Essa convenção facilita a organização do histórico do projeto e a automação de ferramentas de versionamento.
