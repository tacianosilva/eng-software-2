## GitFlow
O git branches é uma ferramenta do sistema permitir que vários desenvolvedores trabalhem no único projeto simultaneamente sem afetar a sua branch principal, e sem interferir no trabalho uns dos outros. Já o pull request é uma solicitação pull que permitir mesclar as alterações conforme do código principal.

## Versionamento Semântico
É um padrão para versionar o software de uma forma previsível, por causa do impacto das mudanças para quem consome o sistema. 

## Convencional Commtis
O convencional commtis é uma convenção de formatar mensagem de commit de forma organizada e padronizada. Ele possui um conjunto de regras objetivas para como as mensagens devem ser escritas, facilitando a leitura e automação de tarefas.
