## GitFlow, Branches e Pull Requests

No Git, as **branches** (ramificações) permitem que desenvolvedores trabalhem em novas funcionalidades isoladamente, sem alterar ou quebrar o código principal do sistema. 

O **GitFlow** é um modelo de trabalho estruturado para o Git. Ele define que o projeto deve ter duas branches principais (a `main`, que guarda o código de produção, e a `develop`, onde o código em desenvolvimento é integrado). Quando precisamos criar algo novo, abrimos branches de suporte (como `feature` para novas opções, `hotfix` para correção urgente de bugs em produção, e `release` para preparar lançamentos).

Um **Pull Request** (ou Merge Request) é o pedido que fazemos no GitHub para pegar o código que terminamos na nossa branch isolada e integrá-lo (fazer o merge) em uma branch principal. Ele cria um ambiente onde a equipe pode revisar o código, discutir mudanças e aprovar a tarefa antes que ela vire parte oficial do projeto.
