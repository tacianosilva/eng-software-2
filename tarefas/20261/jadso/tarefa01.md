## 🌿 Git, Branches e Pull Requests (GitFlow)

O GitFlow é uma estratégia de organização de branches que facilita o desenvolvimento em equipe.

Principais branches:

* **main:** versão estável do sistema
* **develop:** versão em desenvolvimento
* **feature:** novas funcionalidades

Pull Requests são utilizados para revisar o código antes de integrá-lo ao projeto principal, garantindo qualidade e controle das alterações. No GitFlow, novas funcionalidades são desenvolvidas em branches do tipo feature e posteriormente integradas à branch develop através de Pull Requests.

---

## 🔢 Versionamento Semântico

O versionamento semântico segue o padrão:

MAJOR.MINOR.PATCH

* **MAJOR:** mudanças incompatíveis
* **MINOR:** novas funcionalidades compatíveis
* **PATCH:** correções de bugs

Exemplo: 1.0.1

---

## 📝 Conventional Commits

Conventional Commits é uma padronização para mensagens de commit.

Principais tipos:

* `feat:` nova funcionalidade
* `fix:` correção de bug
* `docs:` documentação
* `refactor:` melhoria de código

Essa padronização facilita a leitura do histórico e automações no projeto.
