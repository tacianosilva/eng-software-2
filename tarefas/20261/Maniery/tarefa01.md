# c) Framework

## Django

O Django é um framework web desenvolvido em Python que segue o padrão MVT (Model-View-Template). Seu principal objetivo é acelerar o desenvolvimento de aplicações web seguras e escaláveis.

Entre seus principais recursos estão:

* Sistema de autenticação integrado;
* Painel administrativo automático;
* ORM (Object Relational Mapper);
* Proteção contra ataques comuns da web;
* Estrutura organizada para desenvolvimento de aplicações complexas.

O Django é amplamente utilizado em sistemas corporativos e aplicações que exigem rapidez no desenvolvimento sem abrir mão da segurança e da qualidade do código.

# f) Branches e Pull Requests

O GitFlow é uma estratégia de organização de branches utilizada em projetos que adotam Git. Seu objetivo é estruturar o fluxo de desenvolvimento, facilitando o trabalho colaborativo e o gerenciamento de versões.

As principais branches são:

* main: contém versões estáveis do sistema;
* develop: reúne funcionalidades em desenvolvimento;
* feature: utilizada para criação de novas funcionalidades;
* release: utilizada para preparação de novas versões;
* hotfix: utilizada para correções urgentes em produção.

Os Pull Requests (PRs), também conhecidos como Merge Requests, são mecanismos utilizados para solicitar a integração de alterações entre branches.

Eles permitem revisão de código, discussão técnica e validação antes da incorporação das mudanças ao projeto principal.

O uso conjunto de GitFlow e Pull Requests contribui para a organização do projeto, redução de erros e melhoria da qualidade do software.

# g) Versionamento Semântico

O Versionamento Semântico (SemVer) é um padrão utilizado para identificar versões de software por meio do formato:

MAJOR.MINOR.PATCH

Onde:

* MAJOR: alterações incompatíveis com versões anteriores;
* MINOR: adição de funcionalidades compatíveis;
* PATCH: correções de bugs e pequenos ajustes.

Exemplo:

1.4.2

* 1 → versão principal;
* 4 → novas funcionalidades;
* 2 → correções realizadas.

O SemVer facilita o entendimento das mudanças realizadas em cada versão e auxilia desenvolvedores e usuários na gestão de dependências.

# h) Conventional Commits

Conventional Commits é uma convenção para padronizar mensagens de commit. Seu objetivo é tornar o histórico do projeto mais organizado e compreensível.

Alguns exemplos:

* feat: adiciona uma nova funcionalidade;
* fix: corrige um problema;
* docs: altera documentação;
* style: mudanças de formatação;
* refactor: refatoração de código;
* test: criação ou modificação de testes;
* chore: tarefas de manutenção.

Exemplos de commits:

```text
feat: adicionar cadastro de usuários

fix: corrigir erro de autenticação

docs: atualizar documentação do projeto
```

A utilização dessa convenção melhora a rastreabilidade das alterações e facilita a geração automática de changelogs e versões.
