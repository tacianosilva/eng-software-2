## GitFlow: Branches e Pull Request
O GitFlow organiza o fluxo em:
- **Main:** Versão oficial de lançamento.
- **Develop:** Onde a integração das funcionalidades ocorre.
- **Feature branches:** Criadas a partir da develop para tarefas específicas.
**Pull Request (PR):** Processo onde o código proposto é revisado pela equipe antes de ser mesclado à branch principal, garantindo qualidade e evitando quebras no sistema.

## Versionamento Semântico (SemVer)
Baseado no formato `MAJOR.MINOR.PATCH`:
1. **MAJOR:** Quebras de compatibilidade (API alterada).
2. **MINOR:** Novas funcionalidades sem quebrar o que já existe.
3. **PATCH:** Apenas correções de bugs.

## Conventional Commits
Padronização de mensagens para automação e clareza:
- `feat:` Nova funcionalidade.
- `fix:` Correção de erro.
- `docs:` Mudanças apenas na documentação.
Exemplo: `feat(api): adiciona autenticação jwt #Ub-KtuKwloLzoC6K`
