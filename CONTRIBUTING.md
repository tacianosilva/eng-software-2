# Contribuindo com o Repositório

Este documento orienta a contribuição de discentes para o repositório da
disciplina Engenharia de Software II.

## Como contribuir

1. **Crie uma issue** descrevendo o problema ou melhoria que deseja resolver.
2. **Faça um fork** do repositório e crie um branch para sua modificação.
3. **Envie um Pull Request (PR)** para o branch `main` com suas alterações.
4. Seu PR será revisado pelo professor Taciano Morais Silva, que poderá
   aprová-lo ou solicitar ajustes.

Apenas o professor Taciano Morais Silva pode revisar e aprovar PRs.

## Reportando bugs

Abra uma issue descrevendo o problema encontrado, incluindo:
- Descrição clara do comportamento esperado vs. observado
- Passos para reproduzir
- Links ou referências relevantes

## Sugerindo melhorias

Abra uma issue com o título iniciando por `[melhoria]` e descreva a sugestão.

## Pull Requests

- Faça alterações pequenas e focadas — um PR por issue.
- Mantenha o histórico limpo e com mensagens de commit descritivas.
- Atualize o `README.md` se sua alteração impactar a documentação.

## GitFlow

Utilizamos uma variação do GitFlow. Veja a [página sobre GitFlow](docs/doc-gitflow.md)
para mais detalhes.

## Código de Conduta

Este projeto segue o [Contributor Covenant](https://www.contributor-covenant.org/)
versão 1.4. Ao contribuir, você concorda em manter um ambiente aberto e
acolhedor para todos.
